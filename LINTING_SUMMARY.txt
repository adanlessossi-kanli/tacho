================================================================================
WORKFLOW
================================================================================

Before Committing:
  1. npm run lint:fix      (fix linting errors)
  2. npm run format        (format code)
  3. npm run lint          (verify no errors)
  4. git add .
  5. git commit

During Development:
  - Use IDE integration for real-time feedback
  - Fix issues as you code
  - Run npm run format before committing

In CI/CD:
  - npm run lint (fail if errors found)
  - npm run format --check (fail if not formatted)

================================================================================
IDE INTEGRATION
================================================================================

VS Code:
  1. Install ESLint extension
  2. Install Prettier extension
  3. Add to .vscode/settings.json:
     {
       "editor.defaultFormatter": "esbenp.prettier-vscode",
       "editor.formatOnSave": true,
       "editor.codeActionsOnSave": {
         "source.fixAll.eslint": true
       }
     }

WebStorm/IntelliJ:
  1. Settings → Languages & Frameworks → JavaScript → ESLint
  2. Enable ESLint
  3. Settings → Languages & Frameworks → JavaScript → Prettier
  4. Enable Prettier

================================================================================
TROUBLESHOOTING
================================================================================

"Module not found" Error:
  Solution: npm install

"ESLint not found" Error:
  Solution: npm install --save-dev eslint

"Prettier not found" Error:
  Solution: npm install --save-dev prettier

IDE not showing errors:
  Solution: 
    1. Restart IDE
    2. Check extension is enabled
    3. Verify .eslintrc.json exists

"Line too long" Error:
  Solution: Run npm run format

"Unexpected var" Error:
  Solution: Use const or let instead

"Unused variable" Error:
  Solution: Remove it or prefix with _
